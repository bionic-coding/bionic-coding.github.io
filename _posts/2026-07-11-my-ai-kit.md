Everything in this post is swappable — the models, the harnesses, even the editor. The one piece that stays is [crux](https://github.com/idyll/crux), my own layer that ties it all together. I'll get to it near the end, but keep it in mind as you read: the rest of this list is interchangeable parts.

- **Primary model:** [Opus 4.8](https://www.anthropic.com/claude) (for work/code)
- **Alternate:** [GPT-5.6 Sol](https://openai.com) (the backup for when Opus isn't thinking clearly)

## If you only copy one thing

Don't try to replicate all of this at once. [Zed](https://zed.dev) + [Claude Code](https://www.anthropic.com/claude-code) gets you most of the way there with one subscription and one install. Add a second provider later, when you've hit a wall that a second opinion would have caught. Sign up for API access to the other models so that you can use them in the council. And grab [crux](https://github.com/idyll/crux).

## "Editor"

I am calling this a text editor loosely. Traditionally these were used to edit code.

Now it's a combination of an agentic harness and a text editor.

A LOT of people use [Cursor](https://cursor.com). It's a good place to start.

I use [Zed Editor](https://zed.dev) using [ACP](https://zed.dev/acp) (the Agent Client Protocol) to call [Claude Code](https://www.anthropic.com/claude-code), [OpenCode](https://opencode.ai), or [Codex](https://openai.com/codex).

## Models I use

Two bits of jargon before the list:

- **Sol and Terra** are OpenAI's names for GPT-5.6 variants — Sol is the heavyweight, Terra the fast/cheap one. They're model names, not products I've renamed.
- **High / Xhigh** are reasoning-effort settings — how long the model is allowed to think before answering. More effort costs more and takes longer, so I only turn it up where it pays.
- **Pro / Standard** are two tiers for OpenAI models. I don't know why they are making things so confusing for you.

When I say a model is "supervised," I mean it never works alone: the stronger model plans the work, hands pieces to the cheaper one, and reviews what comes back. The cheap model does the typing; the expensive one stays accountable.

- **[Anthropic](https://www.anthropic.com/claude)**
  - [Opus 4.8](https://www.anthropic.com/claude) High — for work/code
  - [Sonnet 5](https://www.anthropic.com/claude) — only used when being supervised by Opus
  - [Fable](https://www.anthropic.com/claude) — very targeted use: it sits on my "council" (below) as a third voice, and I reach for it on open-ended analysis where I want an answer that isn't shaped like Opus's. I don't route production code through it.
- **[OpenAI](https://openai.com)**
  - [GPT-5.6 Sol](https://openai.com) Pro — used for coding. Part of my "council."
  - [GPT-5.6 Terra](https://openai.com) — only used when being supervised by Sol.
- **[Google Gemini](https://gemini.google.com)**
  - [Gemini Pro 3.1](https://gemini.google.com) — part of my "council."
  - [Gemini Flash 3.5](https://gemini.google.com) — multimodal document understanding.
- **[Perplexity](https://www.perplexity.ai)** — used for search, research, and up-to-date information.

## crux

[crux](https://github.com/idyll/crux) is the layer I promised at the top — the reason the rest is swappable. The models and harnesses above are interchangeable; crux is the part that stays. It's a set of skills and agents that runs in every harness, and a few pieces do the heavy lifting:

- **A real project memory.** crux keeps the intent — architecture decisions, research, briefs, a work journal — as plain Markdown any model can read. The docs are the source of truth; the code stays disposable. It's the [manifesto]({{ "/manifesto/" | relative_url }}) turned into a working system.
- **The council.** The "LLM-as-judge" I keep mentioning: crux puts a decision to several models at once — Claude, GPT, and Gemini — and has them deliberate, agree, or dissent, so I'm not stuck with one model's blind spot.
- **Sourced research.** Instead of prompting a model cold, a crux agent writes the research prompt, captures the source, and files it with a citation. Nothing goes in unsourced.
- **Repeatable cycles.** For real work, crux runs a plan → build → review loop with the council gating the plan — the same process every time, not vibes.

## What I'm no longer using

When Fable first came out I was trying to drive with it, but it's too expensive.
Not only is it more expensive than Opus but it also uses more tokens. This combination
is deadly causing my token costs to go up more than 4x.

I still use it tactically, for example in the council or to help with research, 
but not as the primary driver.

## What I'm doing to cut costs

With the launch of GPT-5.6 Sol Pro I have been experimenting with it as a 
replacement for Opus as a driver.
